## 1. 产品概述

### 1.1 产品定位

**GymClaw** 是一款基于 **CrewAI** 框架的对话式 AI 智能助手，采用多 Agent 协作架构，用户只需通过一个聊天框，即可使用自然语言完成各种复杂任务。

### 1.2 一句话描述

> **GymClaw — 对话式 AI 助手**，打开网页就能用，通过自然语言完成调研、绘图、语音合成、数据分析、消息推送等各种任务。

### 1.3 核心设计原则

| 原则 | 说明 |
|------|------|
| **对话优先** | 所有操作通过聊天完成，无需翻配置页面 |
| **零门槛** | 打开即用，默认 Agent + 预设 Key |
| **可扩展** | 技能系统支持内置 + 自定义扩展 |
| **多模型** | 支持 5+ LLM 供应商，Agent 可独立配置模型 |
| **多语言** | 界面支持中英文切换 |
| **智能路由** | 根据文件类型自动切换到合适的模型 |

### 1.4 目标用户

| 用户类型 | 需求 | 使用方式 |
|----------|------|----------|
| **普通用户** | 打开网页就能聊天，不需要了解技术概念 | 直接在聊天框输入 |
| **进阶用户** | 想换模型、创建自定义技能、配置自己的 API Key | 设置页面操作 |
| **开发者** | 想添加新技能、新插件、二次开发 | 按开发者指南扩展 |

### 2.2 核心特性

#### 2.2.1 多 Agent 协作架构

- **主管 Agent（default）**：负责理解用户意图、分发任务、汇总汇报
- **员工 Agent（用户创建）**：各司其职，绑定不同模型和技能
- **@指派机制**：用户通过 `@员工名` 直接指派任务给特定员工
- **FC 调度**：主管通过 Function Calling 自动调度员工和工具

#### 2.2.2 智能对话引擎

- **流式输出（SSE）**：AI 回复实时逐字显示，体验流畅
- **深度思考模式**：自动切换到推理模型（GLM-Z1/DeepSeek-R1/QwQ-32B）
- **联网搜索**：双路径搜索架构，支持原生工具和独立 API
- **上下文管理**：Token 动态裁剪，自动生成摘要，避免超限

#### 2.2.3 智能路由

- **模型能力检测**：自动检测模型是否支持 Function Calling
- **文件类型路由**：上传图片自动切换到视觉模型（GLM-4V-Flash）
- **思考模式切换**：启用深度思考时自动切换到推理模型

### 6.2 技能匹配算法

```python
# 匹配流程
输入: "帮我画一只可爱的猫"
  ↓
1. 遍历所有已启用技能
2. 检查每个触发词是否出现在输入中（不区分大小写）
3. 同义词扩展（"画" → "绘图、绘画、生成图片"）
4. 匹配到的触发词越长，分数越高（score = len(trigger)）
5. 选分数最高的技能执行
6. 无匹配则走通用对话
```

- **FC 能力检测**：测试模型连接时自动检测 Function Calling 支持
