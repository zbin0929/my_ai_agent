## 1. 系统架构概览

```
┌─────────────────────────────────────────────────────┐
│                    Nginx (80/443)                    │
│                   反向代理 + SSL                      │
├──────────────────────┬──────────────────────────────┤
│   Frontend (:3000)   │    Backend (:8000)            │
│   Next.js 16         │    FastAPI + Uvicorn          │
│   React 19           │    SSE 流式输出                │
│   Tailwind CSS 4     │    Python 3.10+               │
├──────────────────────┴──────────────────────────────┤
│                  数据存储（本地文件系统）               │
│   data/sessions/  data/uploads/  data/memory/        │
├──────────────────────────────────────────────────────┤
│              外部 LLM API 服务                        │
│   智谱AI / DeepSeek / 阿里云百炼 / OpenAI / Moonshot  │
└──────────────────────────────────────────────────────┘
```

**技术栈：**
- 前端：Next.js 16 + React 19 + TypeScript + Tailwind CSS 4 + Zustand
- 后端：FastAPI + Uvicorn + SSE-Starlette + OpenAI SDK
- 核心引擎：多 Agent FC 调度 + 深度思考自动切换推理模型 + 简单模式思考过滤 + 技能绑定 + 双路径搜索 + SSRF 防护
- 数据存储：本地文件系统（JSON）
- 通信协议：SSE（Server-Sent Events）流式传输

## 5. Nginx 反向代理配置

> 项目已提供预配置的 Nginx 文件：`deploy/nginx.conf`，可按 [4.7 配置 Nginx 和 Systemd](#47-配置-nginx-和-systemd) 直接使用。
> 以下为详细说明，供自定义修改时参考。

### 5.1 配置文件说明

`deploy/nginx.conf` 包含以下关键配置：

| 路由 | 代理目标 | 特殊配置 |
|------|---------|---------|
| `/` | `127.0.0.1:3000` (前端) | WebSocket 支持 |
| `/api/` | `127.0.0.1:8000` (后端) | 超时 180s |
| `/api/chat/stream` | `127.0.0.1:8000` (后端) | `proxy_buffering off`，超时 300s |
| `/uploads/` | 静态文件直出 | 缓存 7 天 |
| 静态资源 | `127.0.0.1:3000` | 缓存 7 天 |

### 5.2 SSE 流式输出配置（关键）

SSE（Server-Sent Events）需要禁用 Nginx 缓冲，否则会等待全部内容就绪后一次性返回：

```nginx
location /api/chat/stream {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Connection "";

    proxy_buffering off;      # 禁用缓冲（关键）
    proxy_cache off;          # 禁用缓存
    proxy_read_timeout 300s;  # 延长超时
    chunked_transfer_encoding on;
}
```

## 7. Docker 部署（可选）

### 7.1 创建 Dockerfile

在项目根目录创建 `Dockerfile`：

```dockerfile
# === 后端 ===
FROM python:3.13-slim AS backend

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# === 前端构建 ===
FROM node:20-slim AS frontend-builder

WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ .
RUN npm run build

# === 生产镜像 ===
FROM node:20-slim AS production

RUN apt-get update && apt-get install -y --no-install-recommends \
    python3 python3-venv \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

# 复制后端
COPY --from=backend /app /app
RUN python3 -m venv /app/venv && \
    /app/venv/bin/pip install --no-cache-dir -r /app/requirements.txt

# 复制前端构建产物
COPY --from=frontend-builder /app/frontend /app/frontend

# 创建日志目录
RUN mkdir -p /app/logs /app/data

EXPOSE 3000 8000

COPY <<'EOF' /app/start.sh
#!/bin/bash
set -e
source /app/venv/bin/activate
python -m uvicorn api.main:app --host 0.0.0.0 --port 8000 --workers 2 &
cd /app/frontend && npm run start -- -p 3000 &
wait
EOF
RUN chmod +x /app/start.sh

CMD ["/app/start.sh"]
```

### 7.2 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  gymclaw:
    build: .
    container_name: gymclaw
    ports:
      - "3000:3000"
      - "8000:8000"
    env_file:
      - .env
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    restart: always
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

### 7.3 启动 Docker 部署

```bash
# 构建并启动
docker compose up -d --build

# 查看日志
docker compose logs -f

# 停止
docker compose down
```

| 变量名 | 必填 | 说明 |

| `WECOM_WEBHOOK` | 可选 | 企业微信群机器人 Webhook 地址 |
| `WECOM_SECRET` | 可选 | 企微机器人加签密钥 |
| `DINGTALK_WEBHOOK` | 可选 | 钉钉群机器人 Webhook 地址 |
| `DINGTALK_SECRET` | 可选 | 钉钉机器人加签密钥 |

### 8.4 搜索配置

联网搜索采用全局配置，在 `config/config.yaml` 的 `search:` 节中设置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `search.provider` | 搜索引擎 | `zhipu_search` |
| `search.api_key` | 搜索 API Key（支持 `${ENV_VAR}` 环境变量引用） | `${ZHIPU_API_KEY}` |
| `search.env_key` | 备用环境变量名 | `ZHIPU_API_KEY` |
| `search.model` | 搜索使用的模型（仅 Path B） | `glm-4-flash` |

**双路径搜索说明：**
- 当全局搜索 Key 与当前 Agent 的 Key 相同时（Path A），直接注入原生 `web_search` 工具，零额外 API 调用
- 当 Keys 不同时（Path B），调用独立搜索 API 获取结果，注入到对话上下文中

### 8.5 全局默认 API Key

如果所有模型提供商使用同一个 API Key（如全部使用智谱），只需配置 `default_api_key` 即可，无需为每个提供商单独配置环境变量：

```yaml
# config/config.yaml
default_api_key: "${ZHIPU_API_KEY}"   # 全局默认 Key，各提供商会自动回退使用
```

优先级：`Agent 自定义 custom_api_key` > `提供商 env_key 环境变量` > `default_api_key`

### 8.6 内置模型与 API Key 对应关系

系统会根据已配置的 API Key 自动显示可用模型：

| API Key | 可用模型 |
|---------|---------|
| `ZHIPU_API_KEY` | GLM-4-Flash-250414（128K）、GLM-4.7-Flash（200K MoE）、GLM-4-Air-250414（128K）、GLM-Z1-Flash（免费推理）、GLM-Z1-Air（推理）、CogView-4（图片生成）、GLM-4V-Flash（视觉理解） |
| `DEEPSEEK_API_KEY` | DeepSeek Chat、DeepSeek-R1（推理） |
| `DASHSCOPE_API_KEY` | Qwen-Turbo、Qwen-Plus、QwQ-32B（推理） |
| `MOONSHOT_API_KEY` | Moonshot V1、Kimi K2.5（推理） |
| `OPENAI_API_KEY` | GPT-4o、GPT-4o Mini |

## 9. 常见问题排查

### 9.1 后端启动失败

**问题：** `ModuleNotFoundError: No module named 'xxx'`

```bash
# 确认虚拟环境已激活
source venv/bin/activate
# 重新安装依赖
```


### 9.3 SSE 流式输出中断或一次性输出

**开发模式下流式不生效（等回复完才一次显示）：**

Next.js 的 `rewrites` 代理会缓冲 SSE 响应。v4.6.0 已修复：开发模式下流式请求自动直连后端（`http://localhost:8000`），绕过 Next.js 代理。

如需自定义后端地址，在前端创建 `.env.local`：

```bash
# frontend/.env.local
NEXT_PUBLIC_BACKEND_URL=http://your-backend-host:8000/api
```

**生产环境检查 Nginx 配置：**

确保 Nginx 的 SSE 相关配置正确：
- `proxy_buffering off` — 禁用缓冲
- `proxy_cache off` — 禁用缓存
- `proxy_read_timeout 300s` — 延长超时

```bash
# 测试 Nginx 配置
sudo nginx -t
sudo systemctl reload nginx
```

### 9.5 400 错误：模型不存在

**问题：** 简单模式返回 "API 请求失败（状态码 400）：模型不存在"

**原因：** `config/models.json` 中的模型 ID 映射错误，或 `data/agents.json` 配置的模型 ID 与实际 provider 模型不匹配。

**解决方案：**
1. 检查 `config/models.json` 中对应模型的 `model_id` 是否为 provider 实际支持的模型名称
2. 确保 `core/model_router.py` 的 `resolve_model_id()` 正确映射
3. 重启后端服务使配置生效

```bash
# 检查模型配置
cat config/models.json | grep -A3 '"id": "glm-4-7-flash"'

# 重启服务
./stop.sh && ./start.sh
```

### 9.6 跨会话消息泄漏

**问题：** AI 回复从当前会话短暂显示到其他会话

**原因：** 前端全局 messages 状态未在会话切换时清空，streaming 状态未隔离。

**已修复：** v5.11.0-patch1 已添加 `streamingSessionId` 机制和会话切换时立即清空 messages。

## 附录：快速部署清单

```bash
# ===== 方式一：一键脚本（推荐）=====

# 1. 克隆代码到服务器
sudo mkdir -p /opt/ai-assistant
sudo chown $USER:$USER /opt/ai-assistant
git clone https://gitee.com/你的用户名/ai-assistant.git /opt/ai-assistant

# 2. 运行初始化脚本
cd /opt/ai-assistant && sudo bash deploy/setup.sh

# 3. 配置环境变量和域名
sudo -u www-data nano /opt/ai-assistant/.env
sudo nano /etc/nginx/sites-available/ai-assistant  # 替换 YOUR_DOMAIN

# 4. 启动服务
sudo chown -R www-data:www-data /opt/ai-assistant
sudo systemctl enable ai-backend ai-frontend
sudo systemctl start ai-backend ai-frontend
sudo systemctl reload nginx

# ===== 方式二：手动部署 =====

# 1. 安装基础环境
sudo apt update && sudo apt install -y python3 python3-venv nodejs npm nginx git

# 2. 部署代码
cd /opt/ai-assistant && git clone <repo> .

# 3. 安装后端依赖
python3 -m venv venv && source venv/bin/activate && pip install -r requirements.txt

# 4. 安装并构建前端
cd frontend && npm install && npm run build && cd ..

# 5. 配置环境变量
cp .env.example .env && nano .env

# 6. 配置 Nginx 和 Systemd
sudo cp deploy/nginx.conf /etc/nginx/sites-available/ai-assistant
sudo ln -sf /etc/nginx/sites-available/ai-assistant /etc/nginx/sites-enabled/
sudo cp deploy/ai-backend.service deploy/ai-frontend.service /etc/systemd/system/
sudo systemctl daemon-reload

# 7. 启动服务
sudo chown -R www-data:www-data /opt/ai-assistant
sudo systemctl enable --now ai-backend ai-frontend
sudo systemctl reload nginx

# 8. 验证部署
curl http://localhost:8000/api/health
curl http://localhost:3000

# ===== 后续更新 =====
cd /opt/ai-assistant && bash deploy.sh
```
