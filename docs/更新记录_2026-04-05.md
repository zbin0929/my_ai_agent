### 1. 前端：修复跨会话消息泄漏问题

**问题描述**：AI 回复从当前会话短暂显示到其他会话，刷新后消失。

**修复文件**：
- `frontend/src/store/chatStore.ts` - 添加 `streamingSessionId` 到 StreamSlice，防止跨会话写入
- `frontend/src/components/sidebar/Sidebar.tsx` - 切换会话时立即清空 messages 并重置 streaming 状态
- `frontend/src/components/chat/InputBar.tsx` - SSE 回调中设置 streamingSessionId

**核心改动**：
```typescript
// chatStore.ts: 添加 session 级别的流式状态保护
interface StreamSlice {
  streamingSessionId: string | null;
  setStreamingSessionId: (id: string | null) => void;
  // ... guard finalizeStreamMessage and append operations

// Sidebar.tsx: 切换会话时立即清理
const handleSelectSession = async (id: string) => {
  if (isStreaming) {

    setIsStreaming(false);
  }
  setCurrentSession(id);

  // ...
```

### 2. 前端：修复侧边栏标题实时更新

**问题描述**：会话标题在首次对话后不实时更新，需要刷新页面。

**修复文件**：`frontend/src/components/chat/InputBar.tsx`

**修复策略**：
1. 发送首条消息时，立即用用户消息 snippet 设置临时标题
2. SSE `done` 事件后，用 LLM 生成的标题（或 fallback）更新
3. 调用后端 API 保存标题，然后刷新会话列表

```typescript
// 立即更新临时标题
if (shouldAutoTitle && currentSessionId) {
  const snippetTitle = text.length > 30 ? text.slice(0, 30) + "..." : text;
  updateSessionTitle(currentSessionId, snippetTitle);
}

// done 事件后更新最终标题并刷新列表
if (event.type === "done" && shouldAutoTitle) {
  const finalTitle = event.title || text.slice(0, 30) + "...";
  updateSessionTitle(currentSessionId, finalTitle);
  api.sessions.update(...).then(() => api.sessions.list(...));
}
```

### 3. 后端：修复 LLM API 400 错误（模型不存在）

**问题描述**：简单模式返回 "API 请求失败（状态码 400）：模型不存在"。

**根本原因**：`config/models.json` 中的 `glm-4-7-flash` 配置的 `model_id` 错误映射到不存在的 `glm-4-flash`，应使用正确的 `glm-4.7-flash`。

**修复文件**：
- `config/models.json` - 修正模型 ID 映射，启用 `supports_thinking: true`
- `core/model_router.py` - 添加 `resolve_model_id()` 函数，统一模型 ID 解析
- `core/chat_engine.py` - `_stream_llm_real()` 使用解析后的实际模型 ID

**配置修正**：
```json
{
  "id": "glm-4-7-flash",
  "name": "GLM-4.7-Flash",
  "model_id": "glm-4.7-flash",
  "supports_thinking": true,
  "capabilities": ["tool_use"]
}
```

### 4. 后端：支持 GLM-4.7-Flash 思考模式切换

**问题描述**：简单模式和思考模式需要切换模型。

**解决方案**：GLM-4.7-Flash 支持通过 `enable_thinking` 参数在同一模型内切换思考模式。

**API 参数**：
```python
payload = {
    "model": "glm-4.7-flash",
    "messages": messages,
    "stream": True,
    "enable_thinking": True,   # 思考模式
    # "enable_thinking": False,  # 简单模式
}
```

**修复文件**：`core/chat_engine.py`
```python
if enable_thinking:
    payload["enable_thinking"] = True
else:
    payload["enable_thinking"] = False
```

**行为差异**：
- GLM-4.7/GLM-4.7-Flash: 开启后**强制思考**
- GLM-4.6/GLM-4.5: 开启后**自动判断**是否需要思考

### 6. 后端：修复图片生成安全问题

**问题描述**：
1. 图片生成结果展示本地文件路径，存在信息泄漏风险

**修复文件**：`skills/image_generate.py`

### 7. 后端：修复技能调用参数签名问题

**问题描述**：`execute_tool_by_name` 使用 `handler(**arguments)` 传参，但技能函数签名是 `handler(user_input, context)`，导致 `generate_image() got an unexpected keyword argument 'prompt'` 错误。

**修复文件**：
- `skills/__init__.py` - 修复 `execute_tool_by_name` 函数
- `core/chat_engine.py` - 添加 `get_skill_by_tool_name` 调用

**修复内容**：
```python
# 修复前
return handler(**arguments)

# 修复后
user_input = arguments.get("prompt") or arguments.get("text") or arguments.get("url")
context = {"tool_args": arguments, "skill_id": skill_id}
return handler(user_input, context)
```
