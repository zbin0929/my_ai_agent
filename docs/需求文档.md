## 一、项目定位

### 1.1 一句话描述

**GymClaw — 对话式 AI 助手**，用户只需要一个聊天框，通过自然语言完成各种任务。

### 1.2 核心设计原则

| 原则 | 说明 |
|---|---|
| **对话优先** | 所有操作通过聊天完成，不需要翻配置页面 |
| **零门槛** | 打开就能用，默认 Agent + 预设 Key |
| **可扩展** | 技能系统支持内置 + 自定义 |
| **多模型** | 支持多个 LLM 供应商，Agent 可独立配置 |
| **多语言** | 界面支持中英文切换 |
| **智能路由** | 根据文件类型自动切换到合适的模型 |

### 1.3 用户画像

| 用户类型 | 需求 | 使用方式 |
|---|---|---|
| **普通用户** | 打开网页就能聊天，不需要了解任何技术概念 | 直接在聊天框输入 |
| **进阶用户** | 想换模型、创建自定义技能、配置自己的 API Key | 设置页面操作 |
| **开发者** | 想添加新技能、新插件、二次开发 | 按开发者指南扩展 |

### 1.4 设计参考

参考 MaxClaw 的体验：
- 一个对话框搞定一切
- 技能系统（类似技能商店）
- 无需配置，开箱即用

#### 2.1.2 典型用例

**用例 1：通用对话**

```
前置条件：用户已打开 Web 界面
主流程：
  1. 用户在输入框输入"你好"
  2. 系统解析消息（无 @，无技能触发词）
  3. 系统调用 LLM 进行通用对话
  4. AI 流式返回回复，实时显示在聊天区域
预期结果：AI 用自然语言回复问候
```

**用例 2：触发调研技能**

```
前置条件：用户已打开 Web 界面
主流程：
  1. 用户输入"帮我调研一下 AI Agent 的发展趋势"
  2. 系统检测到触发词"调研"
  3. 系统匹配到"主题调研"技能
  4. 系统调用调研技能 handler
  5. 调研技能创建研究员+写手 Agent 执行任务
  6. 返回调研报告
预期结果：显示技能标签"🔍 主题调研" + 调研报告内容
```

**用例 3：指定 Agent 对话**

```
前置条件：用户已将 Agent 改名为"GymClaw"
主流程：
  1. 用户输入"@GymClaw 今天天气怎么样"
  2. 系统解析 @小明，匹配到当前 Agent
  3. 清洗输入为"今天天气怎么样"
  4. 系统匹配技能（无匹配）
  5. 调用 LLM 通用对话
预期结果：AI 以"小明"的身份回复
```

**用例 4：上传图片分析（智能路由）**

```
前置条件：用户已打开 Web 界面，Agent 使用 glm-4-flash 模型
主流程：
  1. 用户上传一张图片并发送"分析一下这张图"
  2. 系统检测到文件扩展名 .jpg，判断需要 "vision" 能力
  3. 检查 Agent 模型 glm-4-flash，只有 ["text"] 能力
  4. 自动查找支持 "vision" 的模型，找到 glm-4v-flash
  5. 使用 glm-4v-flash 处理本次请求
  6. 返回图片分析结果
预期结果：图片分析成功，用户无需手动切换模型
```

**用例 5：智能标题生成**

```
前置条件：用户已打开新对话
主流程：
  1. 用户输入"帮我分析一下 Helios 大模型的启动文件"并上传文件
  2. 系统匹配到文件分析技能，调用 LLM 分析
  3. 流式输出完成后，done 事件携带 AI 生成的标题"Helios启动文件分析"
  4. 前端自动更新侧边栏对话标题
预期结果：侧边栏显示智能生成的标题，而非用户输入的原文截断
```

**用例 6：管道信息展示**

```
前置条件：用户正在对话中
主流程：
  1. 用户发送消息后，系统开始流式输出
  2. 前端在输入框上方显示管道信息徽章：
     - GymClaw（蓝色，当前 Agent）
     - glm-4-flash（绿色，当前模型）
     - 文件分析（橙色，当前技能）
  3. 流式输出完成后，管道信息消失
预期结果：用户清楚当前对话使用的 Agent、模型和技能
```

#### 2.3.4 技能匹配算法

```
输入: "帮我调研一下AI发展趋势"
  ↓
1. 遍历所有已启用技能
2. 检查每个触发词是否出现在输入中（不区分大小写）
3. 匹配到的触发词越长，分数越高（score = len(trigger)）
4. 反向匹配（输入包含在触发词中）得分较低（score = len(input) * 0.5）
5. 选分数最高的技能执行
6. 无匹配则走通用对话
```
