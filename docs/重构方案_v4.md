**技能配置系统（config_schema）：**

每个技能通过 `@register_skill` 装饰器的 `config_schema` 参数声明配置需求：

| 配置类型 | 说明 | 示例 |
|---------|------|------|
| `text` | 文本输入 | 自定义参数 |
| `password` | 密码输入 | API Key（placeholder: "请输入KEY"） |
| `select` | 下拉选择 | 模型选择、图片尺寸 |
| `number` | 数字输入 | 超时时间 |
| `boolean` | 开关 | 是否启用某功能 |

配置优先级：`data/skill_configs.json` > 环境变量（`env_hint`） > `default` 默认值。

配置状态（config_status）：
- `"ok"`：所有必填项已配置（显示 ✅）
- `"missing"`：存在未配置的必填项（显示 ❌）
- `"none"`：无必填配置项（显示 —）

### 4.2 智能路由机制

当用户上传文件时，系统自动检测所需能力并路由到合适的模型：

```
用户上传图片 + 发送消息
       │
       ▼
  ┌─────────────────────┐
  │ 检测所需能力          │  图片 → 需要 "vision"
  │ _detect_required_    │
  │ capabilities(files)  │
  └──────────┬──────────┘
             │
             ▼
  ┌─────────────────────┐
  │ 检查 Agent 模型能力   │  glm-4-flash 只有 ["text"]
  │ _get_model_          │
  │ capabilities()       │
  └──────────┬──────────┘
             │
             ▼ 不满足
  ┌─────────────────────┐
  │ 查找支持能力的模型     │  找到 glm-4v-flash
  │ find_model_with_     │  ["text", "vision", "ocr"]
  │ capabilities()       │
  └──────────┬──────────┘
             │
             ▼
  自动切换到 glm-4v-flash 处理本次请求
```

**关键设计：**
- 能力检测基于文件扩展名
- 路由是临时的，仅影响当前请求，不修改 Agent 配置
- 如果没有找到支持所需能力的模型，回退到 Agent 默认模型
- 自定义模型添加时可选择能力标签

## 七、后端 API 设计

### 7.1 对话接口（SSE 流式）

```
POST /api/chat/stream
Content-Type: application/json

请求体:
{
  "message": "帮我分析一下量子计算",
  "session_id": "sess_xxx",
  "agent_id": "default",
  "files": ["file_id_1"],
  "file_infos": [{"file_id": "file_id_1", "filename": "data.csv", "size": 1024, "type": "spreadsheet", "url": "/uploads/file_id_1"}],
  "enable_thinking": false,
  "enable_search": false
}

响应: SSE 事件流
event: skill
data: {"type": "skill", "content": "主题调研"}

event: thinking
data: {"type": "thinking", "content": "首先需要理解量子计算..."}

event: content
data: {"type": "content", "content": "量子计算在以下领域..."}

event: done
data: {"type": "done", "agent_name": "GymClaw", "model_id": "glm-4-flash", "skill_used": "research", "skill_name": "主题调研", "title": "量子计算前景分析"}
```

## 八、国际化（i18n）

### 8.1 实现方式

前端使用 **Zustand** 管理 i18n 状态，`useI18n` Hook 提供 `t(key)` 函数：

```typescript
const { t, lang, setLang } = useI18n();
// t("welcome") → "你好，我是 AI 小助手" (zh) / "Hi, I am GymClaw" (en)
// setLang("en") → 切换到英文
```

### 8.2 翻译范围

- 侧边栏（新对话、置顶、重命名、分享、删除、设置等）
- 对话区域（欢迎语、输入框占位符、思考过程标签等）
- 设置页面（Agent 管理、模型配置）
- 模型能力标签（文本、视觉、OCR、表格、图片生成）
- 所有按钮和操作反馈

### 8.3 后端 i18n

后端保留了 Python i18n 模块（`i18n/`），用于 system prompt 语言适配。前端 i18n 与后端独立运作。
